# WeChat Publish + Client Settings Design

日期：2026-04-22

## 1. 目标

在当前 `wechat-article-obsidian` 插件中补齐两类能力：

- 将当前 workbench 已渲染完成的公众号预览发布到微信草稿箱
- 在 Obsidian 插件设置中增加可维护的客户配置与公众号配置

本次设计保持以下硬约束：

- 当前打开的 Markdown 文档仍是正文真源
- workbench 仍是工具台，不是第二编辑器
- 发布时不绑定文档客户归属
- 发布时使用当前 workbench 的预览产物，而不是生成第二份正文来源

## 2. 范围

### 2.1 本期范围

- 全局模型设置继续保留在插件设置中
- 新增客户配置管理
- 每个客户配置绑定一个公众号配置
- workbench 顶部增加客户选择器
- 记住上一次选择的客户配置
- 发布时直接使用当前 workbench 预览 HTML
- 发布逻辑参考 `ls-wechat-article` 现有发布实现

### 2.2 非目标

- 文档级客户绑定
- 多公众号二级选择
- 发布前生成导出稿再发布
- `industry`、`target_audience`、`topics`、`blacklist` 参与当前发布校验或改稿
- 发布历史、统计回写、学习回写
- 安全存储、系统钥匙串集成
- 把 `ls-wechat-article` 整个 CLI 或完整工作流搬进插件

## 3. 配置模型

配置分为两层：全局设置与客户配置。

### 3.1 全局设置

全局设置继续承载当前插件已有能力：

- 图片服务商
- 图片 API Key / Base URL / Model
- 图片规划模型开关 / Base URL / API Key / Model
- 默认封面类型
- 默认风格
- 默认创意方向
- 输出目录开关
- 输出目录路径

新增字段：

- `lastSelectedClientId`

`LLM` 配置与客户配置无关，不下沉到客户配置内。

### 3.2 客户配置

每个客户配置包含：

- `id`
- `author`
- `industry`
- `targetAudience`
- `topics`
- `blacklist.words`
- `blacklist.topics`
- `wechat.accountName`
- `wechat.appid`
- `wechat.secret`

字段职责：

- `author`
  - 作为顶部选择器展示名
  - 作为微信草稿发布时的 `author`
- `wechat.accountName`
  - 仅作为公众号凭证记录名和设置页展示辅助字段
- `wechat.appid/secret`
  - 作为微信接口凭证

客户配置与公众号配置保存在插件内部设置数据中，不单独写 Vault 配置文件。

### 3.3 存储风险

由于配置保存在插件内部设置中，`appid/secret` 仍然是本地明文存储。本期接受这个风险，不扩展到安全存储方案。

## 4. Workbench 交互

### 4.1 顶部客户选择器

workbench 顶部新增一个轻量作者标签控件，用于选择当前发布目标客户配置。

控件结构：

- 装饰性用户 icon
- 作者名称
- 选择箭头

显示规则：

- 多账户时显示选择箭头
- 单账户时不显示箭头
- 无配置时显示 `未设置作者`

行为规则：

- 若没有客户配置：
  - 显示 `未设置作者`
  - 点击后跳转插件设置页
- 若只有一个客户配置：
  - 自动选中该配置
  - 发布时直接使用
  - 点击时可进入设置页，但不弹出下拉
- 若有多个客户配置：
  - 默认恢复 `lastSelectedClientId`
  - 若上次选择不存在，则回退到列表第一个可用配置
  - 点击标签后弹出下拉菜单选择账户

这个选择器不是文档绑定器：

- 不写入 Markdown
- 不表示当前文档归属哪个客户
- 不影响正文真源
- 仅决定当前发布所使用的客户配置

### 4.2 发布按钮

点击 `发布` 后不再弹出二次客户选择框。

发布行为依赖顶部当前选择：

- 未配置客户：阻止发布并提示跳设置页
- 已配置且已选中：直接进入发布流程

## 5. 发布输入与数据流

### 5.1 发布输入源

发布执行层读取的是当前 workbench 已渲染完成的公众号预览结果，而不是重新生成一份独立发布稿。

需要区分两个概念：

- Markdown：正文真源
- 当前预览 HTML：发布产物源

发布执行输入包含：

- 当前预览 HTML
- 当前主题 key
- 当前图片结果
- 当前 sourcePath
- 当前客户配置

### 5.2 发布主流程

1. 用户点击 `发布`
2. 插件读取当前顶部选中的客户配置
3. 插件校验：
   - 是否存在客户配置
   - 是否存在 `author`
   - 是否存在 `appid/secret`
   - 当前预览是否已准备完成
4. 发布服务层消费当前预览 HTML
5. 发布服务层上传正文图片到微信 CDN，并替换 HTML 中的图片地址
6. 发布服务层解析封面资源并上传，获取 `thumb_media_id`
7. 发布服务层调用微信草稿创建接口
8. workbench 返回发布结果状态

### 5.3 与真源实现的对齐方式

发布实现参考 `ls-wechat-article` 的以下能力：

- `toolkit/src/cli.ts` 中的 `publish` 流程
- `toolkit/src/wechat-api.js` 中的 token 获取、正文图片上传、封面上传
- `toolkit/src/publisher.ts` 中的草稿创建

插件内不直接复用 CLI 入口作为长期架构，而是复用发布协议和核心执行逻辑。

## 6. 模块边界

### 6.1 设置存储层

职责：

- 读取与保存全局设置
- 读取与保存客户配置
- 持久化 `lastSelectedClientId`

不负责：

- workbench UI
- 微信发布

### 6.2 客户配置管理 UI

职责：

- 在设置页中新增客户配置列表
- 支持新增、编辑、删除客户配置
- 使用结构化表单编辑字段

不负责：

- 发布执行
- 文档内容处理

### 6.3 顶部客户选择器

职责：

- 展示当前客户配置
- 切换发布目标客户
- 无配置时提供进入设置页入口

不负责：

- 修改正文
- 改变预览内容生成逻辑

### 6.4 发布服务层

职责：

- 消费当前预览 HTML 与客户配置
- 完成微信发布 API 链路
- 汇总发布结果与错误

不负责：

- 选题
- 写作
- 质量校验
- 客户画像解释

## 7. 设置页设计

设置页改为顶部 tab 结构，只保留两个一级页签：

- `账户`
- `模型`

### 7.1 账户 tab

账户 tab 用于客户配置管理，支持结构化表单增删改。

字段布局建议：

- 客户基础信息
  - 作者
  - 行业
  - 目标受众
- 话题配置
  - topics 列表
- 黑名单配置
  - blacklist.words 列表
  - blacklist.topics 列表
- 公众号配置
  - 账号
  - AppID
  - Secret

文案使用中文，不暴露 YAML/JSON 原始编辑器。

### 7.2 模型 tab

模型 tab 承载所有全局模型相关设置，不和客户配置混放。

模型 tab 分为两组：

- `LLM 配置`
- `图片生成配置`

#### 7.2.1 LLM 配置

这一组承载当前代码中原先称为“图片规划模型”的那套配置，但在产品语义上统一改称 `LLM 配置`。

原因：

- 当前这组配置调用的是文本模型
- 它用于图片规划、文本理解和后续智能决策
- 它不是图片生成模型

这一组包括：

- 启用 LLM
- Base URL
- API Key
- Model

#### 7.2.2 图片生成配置

这一组承载真正的出图相关配置与默认值：

- 图片服务商
- 图片 API Key
- 图片 Base URL
- 图片 Model
- 默认封面类型
- 默认风格
- 默认创意方向

现有输出目录相关设置继续保留在模型 tab 中的全局设置区域，不下沉到客户配置。

## 8. 错误处理

### 8.1 配置错误

包括：

- 没有客户配置
- 客户配置缺少 `author`
- 客户配置缺少 `appid`
- 客户配置缺少 `secret`

处理方式：

- 阻止发布
- 在 workbench 显示明确缺失项
- 提供跳转设置页入口

### 8.2 预览状态错误

包括：

- 当前没有可发布预览 HTML
- 预览正在刷新且状态不是最新
- 预览引用图片与当前图片状态不一致

处理方式：

- 阻止发布
- 提示预览未准备完成
- 不在发布层静默重新生成整份预览

### 8.3 微信接口错误

包括：

- token 获取失败
- 正文图片上传失败
- 封面上传失败
- 草稿创建失败

处理方式：

- 终止本次发布
- 保留当前 workbench 状态
- 回传阶段性错误信息

本期不做自动重试与失败补偿。

## 9. 验证范围

### 9.1 设置与选择器

- 客户配置可新增、编辑、删除
- 客户配置字段可保存与恢复
- `lastSelectedClientId` 可恢复上次选择
- 无客户配置时显示 `未设置账户`
- 单客户时自动选中
- 多客户时可切换并记住上次选择

### 9.2 发布链路

- 发布读取当前预览 HTML
- 发布使用当前客户配置的 `author`
- 发布使用当前客户配置的 `appid/secret`
- 正文图片可上传并替换 URL
- 封面可上传并带入 `thumb_media_id`
- 草稿创建成功时返回成功状态
- 任一阶段失败时返回明确错误

### 9.3 现有行为不回归

- 主题切换不回归
- 配图生成与重生成不回归
- workbench 不变成第二编辑器
- Markdown 仍是正文真源

## 10. V1 明确不做

- 文档级客户绑定
- 发布时二次弹窗选择客户
- 多公众号二级选择
- 客户画像字段参与自动校验或内容加工
- 发布历史写回
- 统计回填
- 学习回写
- 安全存储
- 完整 CLI 套壳式接入

## 11. 设计结论

本期设计收敛为：

在保持 Markdown 为正文真源的前提下，插件新增一个内部客户配置系统和顶部发布目标选择器；发布时直接消费当前 workbench 已渲染完成的公众号预览 HTML，并参考 `ls-wechat-article` 现有发布实现，把内容发到所选客户对应的微信草稿箱。
